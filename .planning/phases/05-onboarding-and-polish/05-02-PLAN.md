   - Heading: "Your life in weeks" (or similar evocative phrasing)
   - Brief text explaining the 4000-week concept: "From age 18 to 95, life is roughly 4,000 weeks. Each one deserves to be remembered."
   - Date input for birth date (use native HTML `<input type="date">` for simplicity -- no date picker library needed)
   - Label: "When were you born?" with warm, inviting tone
   - "Begin" or "See my weeks" button (disabled until date selected)
   - Use existing shadcn Button component

   **Step 2 -- Revelation + Transition:**
   - After user enters birth date and clicks Begin:
     - Calculate their current age in weeks using `getWeekNumber()` math: `Math.floor(differenceInCalendarDays(today, birthDate) / 7)`
     - Calculate weeks remaining to 4000
     - Show a brief revelation message with motion/react animation (fade in):
       - "You've lived {X} weeks."
       - "You have roughly {4000-X} left."
       - Brief pause (1-2 seconds) for emotional weight
     - Save birthDate to Dexie settings: `db.settings.put({ key: 'birthDate', value: dateString, updatedAt: ISO })`
     - Also save the calculated startWeekNumber (week 1 for this user): `db.settings.put({ key: 'startWeekNumber', value: 1, updatedAt: ISO })`
   - After the pause, transition to the grid by setting `onComplete` callback

   **Styling:**
   - Full screen with `fixed inset-0 z-50`
   - Background: `bg-stone-50 dark:bg-stone-950` (same as grid)
   - Text: warm stone palette, large readable font
   - Use motion/react for step transitions (fade/slide)
   - The revelation numbers should be large and prominent (text-4xl or text-5xl font-light)
   - Minimal, museum-like feel -- no clutter

   **Props:**
   - `onComplete: () => void` -- called when onboarding finishes, parent switches to grid view

2. Update `src/app/page.tsx`:
   - Add `useLiveQuery` import from dexie-react-hooks (already used in other components)
   - Query `db.settings.get('birthDate')` with `useLiveQuery`
   - If birthDate is undefined (still loading from Dexie): show nothing (or a subtle loading indicator)
   - If birthDate setting does not exist (first-time user): render `<OnboardingModal onComplete={...} />`
   - If birthDate setting exists (returning user): render existing grid + card view
   - The `onComplete` callback does not need to do anything -- since `useLiveQuery` is reactive, once birthDate is saved to Dexie in the modal, the query will re-fire and the component will automatically switch to showing the grid

3. Update `src/lib/week-utils.ts`:
   - Add a helper function `getWeeksLived(birthDate: Date): number` that calculates how many weeks the user has lived since birth
   - Implementation: `Math.floor(differenceInCalendarDays(new Date(), birthDate) / 7)`
   - This is used by the onboarding modal for the revelation message

Note: The grid currently hardcodes EPOCH_DATE as the starting point. The birth date from onboarding does NOT change the epoch -- the epoch stays at Week 943 = 2025-07-20 (founder's timeline). The birth date is used purely for the emotional onboarding message ("You've lived X weeks"). The grid continues to show all 4000 weeks from Week 1 onward.
