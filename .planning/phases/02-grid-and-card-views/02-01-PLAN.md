**2. Create `src/lib/grid-utils.ts`** -- Grid layout constants and week state derivation:

```typescript
import { db } from './db';

// Grid layout constants
export const TOTAL_WEEKS = 4000;
export const WEEKS_PER_ROW = 52; // 52 weeks per year row
export const TOTAL_ROWS = Math.ceil(TOTAL_WEEKS / WEEKS_PER_ROW); // ~77 rows

// Week states for cell coloring
export type WeekState = 'empty' | 'has-captures' | 'synthesized' | 'current' | 'future';

/**
 * Determine the visual state of a week cell.
 * Priority: current > synthesized > has-captures > future > empty
 */
export function getWeekState(
  weekNumber: number,
  currentWeekNumber: number,
  hasEntries: boolean,
  hasSynthesis: boolean
): WeekState {
  if (weekNumber === currentWeekNumber) return 'current';
  if (weekNumber > currentWeekNumber) return 'future';
  if (hasSynthesis) return 'synthesized';
  if (hasEntries) return 'has-captures';
  return 'empty';
}

/**
 * Get the approximate year label for a row in the grid.
 * Row 0 = the year containing Week 1.
 * Each row = 52 weeks ~ 1 year.
 */
export function getYearForRow(row: number, week1Year: number): number {
  return week1Year + row;
}

/**
 * Convert row and column to week number.
 * row=0, col=0 => Week 1
 */
export function rowColToWeekNumber(row: number, col: number): number {
  return row * WEEKS_PER_ROW + col + 1;
}

/**
 * Convert week number to row and column.
 */
export function weekNumberToRowCol(weekNumber: number): { row: number; col: number } {
  const index = weekNumber - 1;
  return {
    row: Math.floor(index / WEEKS_PER_ROW),
    col: index % WEEKS_PER_ROW,
  };
}

/**
 * Get the row that contains a given week number (for auto-scrolling).
 */
export function getRowForWeek(weekNumber: number): number {
  return Math.floor((weekNumber - 1) / WEEKS_PER_ROW);
}
```

Key design decisions:
- 52 columns per row (one row ~ one year of life). This makes the grid naturally read as "years on Y axis, weeks on X axis."
- Week 1 is at row 0, col 0. Week 4000 is at the bottom.
- WeekState enum drives cell color. "current" has highest priority so it always stands out.
- "future" state differentiates weeks beyond the current week (they should appear distinctly from "empty" past weeks).

Implementation details:

a) **Data fetching for cell states:**
   - Use `useLiveQuery` to fetch all entry weekIds and synthesis weekIds:
     ```typescript
     const entryWeekIds = useLiveQuery(async () => {
       const entries = await db.entries.orderBy('weekId').uniqueKeys();
       return new Set(entries as string[]);
     });
     const synthesisWeekIds = useLiveQuery(async () => {
       const syntheses = await db.syntheses.toArray();
       return new Set(syntheses.map(s => s.weekId));
     });
     ```
   - These are lightweight index scans, not full table reads. The Set lookup is O(1) per cell.

b) **Row virtualization with TanStack Virtual:**
   - Use `useVirtualizer` with:
     - `count: TOTAL_ROWS` (~77 rows)
     - `getScrollElement: () => scrollRef.current`
     - `estimateSize: () => ROW_HEIGHT` (calculate based on cell size + gap)
     - `overscan: 5` (render 5 extra rows above/below viewport)
   - The virtualizer manages ROW virtualization. Within each virtual row, render 52 `<WeekCell>` components. Since only ~10-15 rows are visible at once, this means ~520-780 cells max in the DOM -- well under performance limits and the overscan provides smooth scroll. The cells within a row are simple divs, not individually virtualized.

c) **Grid layout:**
   - Outer container: `ref={scrollRef}`, overflow-y scroll, fixed height (calc to fill viewport)
   - Each virtual row: `flex` container with 52 cells, gap-[1px] or gap-0.5
   - Left margin: year label column showing the year number for that row. Year labels use `getYearForRow()`. Use absolute positioning or a fixed-width left column.
   - Desktop: cells are 10-12px with 1px gap. Total width ~52*(12+1) = ~676px. Centered on page.
   - Mobile: cells are 6-8px with 0.5px gap. Total width ~52*(7+0.5) = ~390px. Fits most phones.
   - Use CSS custom properties or Tailwind responsive classes: `w-2.5 h-2.5 sm:w-3 sm:h-3`

d) **Auto-scroll to current week on mount:**
   ```typescript
   const currentWeek = getCurrentWeek();
   const currentRow = getRowForWeek(currentWeek);

   useEffect(() => {
     // Wait for virtualizer to be ready, then scroll to current week row
     // Center the current row in the viewport
     rowVirtualizer.scrollToIndex(currentRow, { align: 'center' });
   }, []);
   ```

e) **Week 1 year calculation:**
   Week 1 starts at 2007-07-01 (from Phase 1 decisions). The year for row 0 is 2007. Use `getWeekBoundaries(1).start` to derive the year dynamically:
   ```typescript
   const week1Year = parseInt(getWeekBoundaries(1).start.substring(0, 4));
   ```

f) **Interaction:**
   - onClick on a cell: `useUIStore.setSelectedWeekNumber(weekNumber)`. This will be consumed by the card view in Plan 02.
   - onHover: `useUIStore.setHoveredWeekNumber(weekNumber)` for showing a tooltip/info bar.

g) **Info bar at top or bottom:**
   Show a small info area displaying the hovered week's details: "Week {N} | {start} - {end}". When not hovering, show current week info. This gives immediate context as users scan the grid.

h) **Warm museum aesthetic:**
   - Background: warm off-white `bg-stone-50` or `bg-amber-50/30`
   - Year labels: `text-stone-400 text-xs font-mono`
   - Subtle top heading: "Your 4000 Weeks" in a soft serif or the Geist font at reduced weight
   - Cells have no harsh borders -- the gap between cells provides visual separation
