<objective>
Build the week card detail view with a 3D flip animation between front and back faces. Clicking a week cell in the grid opens the card; the front shows the week number (with graceful degradation when no AI headline exists), and the back shows raw captures with timestamps.

Purpose: The card is the second core UI element after the grid. It represents a single week as a tangible, flippable artifact -- like a museum exhibit card. The flip interaction creates a sense of discovery: front = identity of the week, back = the raw content. This completes the browse-then-view flow that makes the grid useful.

Output: A flippable week card that opens as a modal/overlay from the grid, with a front face (week number + optional headline) and a back face (timestamped captures).
</objective>

**3. Create `src/components/card/WeekCard.tsx`** -- Card container with flip animation:

```typescript
'use client';

import { useState, useCallback } from 'react';
import { motion, AnimatePresence } from 'motion/react';
import { useUIStore } from '@/stores/ui-store';
import { CardFront } from './CardFront';
import { CardBack } from './CardBack';
```

IMPORTANT: Import from `motion/react` (NOT `framer-motion`). The library was rebranded. The import path for React components is `motion/react`.

Props: `weekNumber: number`

State: `isFlipped: boolean` (local state, starts false = front showing)

Implementation:

a) **Modal/Overlay approach:**
   The card opens as a centered overlay on top of the grid. Use a semi-transparent backdrop (`bg-black/40`) to dim the grid. The card itself is a fixed-size container centered on screen.

b) **Card container:**
   - Fixed size: `max-w-md w-full` (constrained width), `min-h-[400px]` (enough for content)
   - Perspective for 3D effect: The parent wrapper needs `perspective: 1000px` in CSS.
   - Two faces (front and back) positioned absolute, backface-visibility hidden.

c) **3D Flip animation with Motion:**
   Use `motion.div` for the card container that rotates on the Y axis:

   ```typescript
   // Flip container approach
   <div style={{ perspective: 1000 }}>
     <motion.div
       animate={{ rotateY: isFlipped ? 180 : 0 }}
       transition={{ duration: 0.6, type: 'spring', stiffness: 200, damping: 30 }}
       style={{ transformStyle: 'preserve-3d' }}
       className="relative w-full min-h-[400px]"
     >
       {/* Front face */}
       <div style={{ backfaceVisibility: 'hidden' }} className="absolute inset-0">
         <CardFront weekNumber={weekNumber} />
       </div>
       {/* Back face */}
       <div style={{ backfaceVisibility: 'hidden', transform: 'rotateY(180deg)' }} className="absolute inset-0">
         <CardBack weekNumber={weekNumber} />
       </div>
     </motion.div>
   </div>
   ```

   The spring transition gives a satisfying, physical flip feel (museum card flipping over).

d) **Flip trigger:**
   Clicking the card flips it: `onClick={() => setIsFlipped(!isFlipped)}`.
   Add a small visual cue: "Click to flip" or a flip icon on first view (fades after first flip).

e) **Close button:**
   An "X" button or click-outside-to-close for the overlay:
   - Close button: absolute positioned top-right of the overlay, `z-50`, using lucide-react `X` icon
   - On close: `useUIStore.getState().setSelectedWeekNumber(null)`
   - Click on backdrop also closes

f) **Entry/exit animation:**
   Use `AnimatePresence` + `motion.div` for the overlay:
   - Enter: fade in backdrop, scale-up card from 0.95 to 1.0
   - Exit: fade out backdrop, scale-down card

g) **Styling -- warm museum feel:**
   - Card background: `bg-white dark:bg-stone-900`
   - Rounded corners: `rounded-xl`
   - Shadow: `shadow-2xl` for elevation
   - Border: `border border-stone-200 dark:border-stone-700`
   - The overall feel should be like picking up a card in a museum gift shop

h) **Keyboard navigation:**
   - Escape key closes the card
   - Space/Enter flips the card
   - Add a `useEffect` with keydown listener
